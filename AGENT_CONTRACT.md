# חוזה סוכנים מרכזי (Agent Contract)

## מטרה
קובץ זה נועד לשמש כמקור האמת וההתנהגות המחייב לכל סוכן אוטונומי הפועל בפרויקט זה. 
כל סוכן חייב לקרוא קובץ זה בתחילת ריצתו, ולעדכן אותו (ו/או את הקבצים המקושרים אליו) בסיום ריצתו או בעת גילוי מידע חדש מהותי.

## חוקי יסוד
1. **קריאה חובה**: בתחילת כל משימה, על הסוכן לקרוא את `AGENT_CONTRACT.md`, `arch_state.md` ולקרוא/לעדכן את `SESSION_LOG.md`.
2. **תיעוד**: כל פעולה משמעותית צריכה להיות מתועדת ב-`SESSION_LOG.md`. אין לבצע שינויים ארכיטקטוניים מבלי לעדכן את `arch_state.md`.
3. **עבודה לפי תהליכים**: ביצוע משימות ייעשה רק לאחר עיון בתיקיית `workflows` ובחירת ה-workflow המתאים.
4. **עקרונות עיצוב**: יש להקפיד על עיצוב מודרני, נקי, מיקרו-אינטראקציות (Glassmorphism, תנועה חלקה) ופלטת צבעים שנקבעה מראש (ראה `arch_state.md`).
5. **בדיקות ואישור איכות**: כל רכיב או עמוד שנוסף חייב להיבדק (ויזואלית ופונקציונלית) טרם סיום המשימה. סוכן יפעיל את שרת הפיתוח וישתמש בכלי /browser כדי לוודא תקינות.
6. **ניהול גרסאות (Git)**: סוכנים מחויבים תמיד לעדכן ולנהל את מאגר הגיט. בסיום כל משימה או שינוי, יש לבצע הוספה (git add), כתיבת הודעת commit ברורה באנגלית, ודחיפה (git push) לענף המרכזי.

## סטנדרט הידע
- טכנולוגיות הליבה: React, Vite, CSS טהור (או Tailwind במידה ואושר).
- דמו מקומי: אין שרת Backend אמיתי. כל ניהול המצב (State) נעשה ב-Client Memory (RAM). אובדן נתונים בריענון הוא התנהגות מתוכננת לשלב זה.

## חובת עדכון
בכל מקרה של:
- הוספת ספרייה חדשה
- שינוי בנתיבי הניווט (Routing)
- שינוי במבנה התיקיות
חובה על הסוכן לעדכן את `arch_state.md` בהתאם.
