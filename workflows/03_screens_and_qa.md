# Workflow: מימוש המסכים והבדיקות (Screens Implementation & QA)

## מטרה
בניית 7 המסכים המאופיינים, אינטגרציה ביניהם ובדיקה מעמיקה כוללת אוטומציה של /browser.

## שלבי ביצוע
1. יצירת 7 הקומפוננטות בתיקיית `src/screens`. כל מסך יהיה אלמנט של `100dvh` עם מאפיין גלילה תואם.
2. מימוש פונקציונליות אינטראקטיבית בכל מסך: הקלפים המרחפים במסך 2, המתגים במסך 3, וסליידר האישור במסך 5.
3. הזרקת פונקציות גלילה כך שפעולה חיובית תגרום ל-`scrollIntoView({ behavior: 'smooth' })` למסך הבא.
4. **בדיקת הסוכן**:
   - הסוכן יפעיל `npm run dev`.
   - הסוכן יפעיל את כלי `/browser` ויכנס ל-`http://localhost:5173`.
   - הסוכן ייקח צילומי מסך מהדפדפן באמצעות הפקודה המתאימה.
   - הסוכן יעבור מסך-מסך באופן אוטומטי (לחיצה על כפתורים) ויוודא שהגלילה אכן מתבצעת ללא שגיאות ב-Console.

## וידוא ותוצר
- חווית משתמש חלקה, עמידה במבחן הדפדפן, קוד ללא Error.
- תיעוד סופי ב-`SESSION_LOG.md` והצגת לינק עובד לשרת המקומי לטובת המשתמש.
